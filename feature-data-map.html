데이터 담당 요약 — 누가 뭘 만들어야 하는가

대표 가 만들어야 하는 데이터

  • 공정별 시공 가이드 (시공 순서, 주의사항, 실패 포인트)
  • 공정-자재 매핑 ("방수 = 방수액 + 부직포 + ...")
  • 자재별 단위 소요량 (1㎡당 몇 개/리터)
  • 자재 호환성 매트릭스 (OK/NG + 사유 + 대체 추천)
  • 스타일/등급별 자재 변형 (모던/클래식 × 경제/표준/프리미엄)
  • 공간 유형별 표준 면적 (평형대별)
  • 아파트별 비용 추정 (셀프/반셀프/업체 3단계)
  • 공정별 적정 단가 + 인건비 기준
  • 표준 공정 순서 + 소요일 + 양생 시간
  • "빌드어스 보증" 자재 지정
  • 로스율 (공정/자재별 여분 비율)

유통 이 만들어야 하는 데이터

  • 자재 카탈로그 (품명, 규격, 브랜드, 제조사, 카테고리)
  • 가격 정보 (도매가, 업체가, 소비자 판매가)
  • 공급사 정보 (입점 업체, 리드타임, 최소 주문, 반품)
  • 배송 정보 (무게, 부피, 배송 방법, 예상일)
  • 자재별 규격 (600×600, 300×600 등)
  • 판매 단위 (박스 장수, 캔 용량 등)
  • 샘플 목록/규격/비용/재고
  • 자재 가격 동향 (주기적 업데이트)

어시 가 만들어야 하는 데이터

  • 인터넷 최저가 조사 (쿠팡/네이버 소비자가)
  • 상품 사진 촬영
  • 상품 설명 작성
  • 체크리스트 → Q&A 변환 (대표 감수)
  • SEO 키워드 조사
  • SEO 콘텐츠 원고 작성
  • 아파트 단지 기본 정보 조사
  • FAQ 데이터 축적 (CS에서 수집)

설계 로 결정할 것

  • 레벨 기준 + 혜택 구조
  • 공정 태그 분류 체계
  • → 현장 데이터 불필요, 기획으로 결정
데이터 재사용 관계:
대표의 "시공 가이드" → 마켓플레이스(1) + 챗봇(6) + 유튜브(11) 에서 동시 사용
대표의 "공정-자재 매핑" → 비용 분석기(2) + 수량 계산기(3) + 챗봇(6) 에서 동시 사용
대표의 "호환성" → 마켓플레이스(1) + 호환성 체커(7) + 챗봇(6) 에서 동시 사용
유통의 "카탈로그+가격" → 마켓플레이스(1) + 비용 분석기(2) + 견적 비교(5) + 챗봇(6) 에서 동시 사용
